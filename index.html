Welcome to the Accessible JavaScript Workshop!

This is a hands-on workshop where you'll learn to identify and fix common accessibility issues in JavaScript-powered websites.

Challenge: This website is currently covered by a black overlay. Your mission is to navigate using only screen readers and keyboard controls to find and activate the "Turn off overlay" button. Along the way, you'll discover various accessibility barriers that need to be addressed.

Workshop Instructions

Right now, you can only interact with this site using assistive technologies like screen readers. This simulates the experience of users who rely on these tools daily.

Your Goal

Navigate through this page to find the "Turn off overlay" button. When you click it, a confirmation modal will appear. You'll need to navigate to that modal and confirm your choice to reveal the visual interface.

What You'll Learn

  • Importance of semantic HTML structure
  • Proper keyboard navigation patterns
  • ARIA labels and live regions
  • Focus management in modals
  • Accessible dropdown menus
  • Form accessibility best practices

Note: As you work through this workshop, you'll be adding proper accessibility features to make this site usable for everyone.

About This Workshop

Modern JavaScript has revolutionized web development, enabling dynamic, interactive experiences. However, with great power comes great responsibility - ensuring these experiences remain accessible to all users.

This workshop uses a "reverse engineering" approach where you start with an inaccessible site and progressively fix issues until it becomes fully accessible.

You'll work with real-world scenarios including dropdown navigation, modal dialogs, form interactions, and dynamic content updates.
